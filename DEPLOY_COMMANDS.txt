賃貸管理アプリ 更新・デプロイ手順（PowerShell）
================================================

1. Supabase SQL Editorで未実行のmigrationを番号順に実行

supabase/migrations/003_improve_settings_and_codes.sql
supabase/migrations/004_remove_actual_monthly_rent.sql
supabase/migrations/005_operations_tasks_documents.sql
supabase/migrations/006_automatic_billing.sql
supabase/migrations/007_reminder_and_relation_cleanup.sql
supabase/migrations/008_contract_initial_fees.sql
supabase/migrations/009_contract_termination_reason.sql
supabase/migrations/010_contract_and_property_refinement.sql
supabase/migrations/011_allow_overpayment.sql
supabase/migrations/012_remove_legacy_payment_cap.sql

※ SQLファイルの内容をSupabase Dashboard > SQL Editorへ貼り付けて実行します。
※ 実行済みの番号は飛ばし、未実行分だけを番号順に適用します。
※ 010は保証会社・振込口座マスタ、想定売却価格、契約区分整理、開始月日割りを追加します。
※ 011は既存請求の再計算で実入金を維持するため、過入金を許可します。
※ 012は既存DBに残った旧入金上限制約を削除します。

2. プロジェクトへ移動

Set-Location "C:\Users\f-yuj\dev\rental-management-app"

3. ローカル検証

npm install
npm run lint
npm run typecheck
npm run test
npm run build

4. GitHubへpush

git status
git add README.md DEPLOY_COMMANDS.txt app components lib types supabase/migrations
git commit -m "Refine contracts and property metrics"
git push origin main

5. Fly.ioへデプロイ

$supabaseUrl = ((Get-Content .env.local | Where-Object { $_ -match '^NEXT_PUBLIC_SUPABASE_URL=' }) -split '=', 2)[1].Trim()
$supabaseAnonKey = ((Get-Content .env.local | Where-Object { $_ -match '^NEXT_PUBLIC_SUPABASE_ANON_KEY=' }) -split '=', 2)[1].Trim()
fly deploy --build-arg "NEXT_PUBLIC_SUPABASE_URL=$supabaseUrl" --build-arg "NEXT_PUBLIC_SUPABASE_ANON_KEY=$supabaseAnonKey"

6. デプロイ確認

fly status
fly logs --app rental-management-app
Invoke-WebRequest -UseBasicParsing "https://rental-management-app.fly.dev/login" | Select-Object StatusCode

公開URL
https://rental-management-app.fly.dev/

注意
・Supabaseのanon keyやservice role keyをこのファイルやGitへ記載しないでください。
・画面で新しいDB項目を使用する前に、必ず未実行のmigrationを012まで実行してください。
